End-to-End 시퀀스 11 STEPS

단계별 설명 DETAIL

1

import 호출

Lillys서버→서버

POST /api/partner/import · HMAC-SHA256 서명 + photos[] + meta. Idempotency-Key 필수.

2

검증 + 티켓 발급

100p

서명/ts/idem 검증 → partner_imports INSERT → ticket(SHA256 저장, 30분 TTL) 생성.

3

202 응답

100p

{ ticket, redirect_url, estimated_ready_seconds } 반환. 사진 fetch 는 백그라운드.

4

비동기 사진 fetch

100p · async

Lillys CDN 에서 photos[].url 다운로드 → Storage 적재 → 자동 레이아웃 (polaroid/collage 등).

5

사용자 redirect

Lillys

HTTP 302 → https://100pbooks.com/p/lillys/start?ticket=…. 사용자는 별다른 입력 없이 진입.

6

ticket 사용

User

브라우저가 100p Books 의 ticket 검증 페이지에 접속. 토큰 1회 사용 + 만료 검증.

7

세션 발급 + 에디터

100p

Set-Cookie (Supabase JWT) + 302 → /editor/[projectId]. 자동 레이아웃된 상태에서 시작.

8

편집

User

Fabric.js 에디터에서 내지 + 표지 자유 편집. 사진 추가/삭제, 텍스트, 스티커.

9

컨펌 + 결제

User100p

책 사이즈/수량/배송지 확정 → TossPayments confirm. address 는 import 데이터로 prefill.

10

PDF + 인쇄 발주

100p · async

결제 성공 즉시 300dpi PDF 빌드 (표지 + 내지) → 인쇄소 발주 큐 적재.

11

webhook 1차

100p서버→서버

POST {lillys_webhook_url} · event: order.status_changed · status=paid.

12

webhook 반복

100p

in_production / shipped (운송장 포함) / delivered / cancelled / refunded 마다 동일 webhook.

13

Lillys 복귀

User

주문 성공 페이지에서 "Lillys 로 돌아가기" 클릭 → callback_url 로 이동.

API 명세 — 4개 엔드포인트 API

POST /api/partner/import Lillys → 100p
사진 묶음 + 메타 import. HMAC + Idempotency-Key 필수.
  • req: photos[1..100] · partner_user_email · title · book_size_hint · address?
  • res 202: { import_id, ticket, redirect_url, estimated_ready_seconds }
  • err: 400 INVALID_BODY · 401 INVALID_SIGNATURE · 413 TOO_MANY_PHOTOS
GET /api/partner/import/[id] Lillys → 100p
진행도 폴링 / 멱등 조회.
  • res: { status: pending|importing|ready|failed, photos_imported, project_id }
POST {lillys_webhook_url} 100p → Lillys
주문 상태 변경 통지. 멱등(event_id) + 재시도(1m/5m/30m/2h/12h).
  • event: order.status_changed
  • data: { order_id, import_id, status, amount, tracking? }
  • 헤더: X-100p-Signature (HMAC) · X-100p-Event-Id · X-100p-Ts
POST /api/partner/lookup Lillys → 100p
CS 화면용 — partner_user_id 로 진행 중 주문/프로젝트 조회.
  • res: { orders[], projects[] }

보안 / 멱등 / 운영 SECURITY

위협방어

Ticket 탈취HTTPS only · 30분 TTL · ticket_hash 저장(평문 미저장)
